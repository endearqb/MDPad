# MDPad 更新说明

- 生成时间：2026-04-20 08:00（Asia/Shanghai）
- 文档类型：仓库内更新记录

## 更新概览

本轮更新聚焦 HTML 预览中的 SVG 编辑能力，目标是把原本偏字段面板式的编辑流程升级成更接近画布操作的轻量编辑体验，同时保持现有源码 patch 保存机制不变。

## 本轮重点改进

### 1. SVG 编辑入口切换为画布式编辑器

- `HtmlPreview` 已接入 `SvgTextCanvasEditor` 作为当前实际使用的 SVG 编辑界面。
- 保留原有 iframe 消息桥与最小 patch 写回策略，不改整体宿主协议。
- 选中、草稿、预览同步与最终应用保存现在由同一套会话状态串联。

### 2. 文字与直线编辑能力增强

- `text` / `tspan` 支持双击进入文字编辑，并保留侧栏文本框。
- 新增 `text-anchor`、`font-family` 等高价值文字属性编辑。
- `line` 支持端点拖拽，除数值字段外可直接在画布上调整两端位置。
- 连线类样式新增箭头、虚线、线帽、折角等控制项。

### 3. 路径与点集编辑能力补齐

- `path` 现已支持 `d` 路径数据文本编辑与基础合法性校验。
- `polyline` / `polygon` 现已支持 `points` 文本编辑。
- 画布侧新增点句柄拖拽，以及 `polyline` / `polygon` 的插点、删点操作。
- 非法 `path` 数据会阻止应用，避免 UI 能改但源码保存无效。

### 4. `text` / `tspan` 兼容性更完整

- 父 `text` 不再因为包含 `tspan` 被整体排除出编辑列表。
- 当前模型允许保留父 `text` 的定位/样式编辑，同时对子 `tspan` 单独编辑文本与坐标。
- 保存时继续沿用 locator 驱动的局部 patch，尽量不破坏原有 SVG 结构。

### 5. 回归测试与构建验证已补齐

- 已为 SVG patch、预览提取、画布编辑器交互、`HtmlPreview` 会话同步补充回归测试。
- 定向 `vitest` 通过，覆盖 `path d`、`points`、高级样式、`tspan` 层级与路径校验。
- `tsc --noEmit` 与 `pnpm build` 已通过。

## 当前边界

当前边界已较上一版明显收缩，下面是本次补充实现后的最新状态。

## 补充更新（2026-04-20 09:45）

### 6. 会话内多选、撤销重做与视口控制已补齐

- `SvgTextCanvasEditor` 已引入显式会话状态，覆盖当前 SVG 编辑会话内的 `past / present / future` 历史栈。
- 支持 `Undo / Redo` 按钮与快捷键，历史粒度围绕“用户完成一次意图”记录。
- 画布侧新增多选、框选、批量移动与视口缩放/平移能力，编辑器不再局限于单元素字段编辑。

### 7. 连接线自动吸附与正交路由已落地

- `line`、`polyline` 和可解析为正交折线的 `path` 现在都能进入连接线模式。
- 端点拖拽会自动寻找最近图形边缘锚点并吸附。
- 图形移动后，已绑定的连接线会在同一 SVG 会话内按正交逻辑重新计算几何。
- 最终保存仍保持最小 patch：`line` 写端点，`polyline` 写 `points`，正交 `path` 写 `d`。

### 8. 贝塞尔控制点编辑已补齐常见命令

- 非路由 `path` 已支持 `M/L/C/Q/S/T/Z` 的锚点和控制柄可视编辑。
- 支持拖锚点、拖控制柄、双击插点、键盘删除锚点。
- 含 `A` 弧线的路径继续保留文本级 `d` 编辑与高亮，不强行开放不稳定的可视手柄。

### 9. 高阶 SVG 回归测试已补上

- 新增编辑器交互测试，覆盖多选批量样式应用与会话内撤销重做。
- 新增几何纯逻辑测试，覆盖连接线路由候选判定、正交回写、贝塞尔句柄更新与路径插删点。
- `tsc --noEmit`、定向 `vitest` 和 `pnpm build` 均已通过。

## 最新边界

- 当前自动路由仍定位于同一 SVG 内的正交流程图连线，不做全局拥塞求解。
- `A` 弧线仍以文本级编辑为主，不提供可视手柄。
- 批量排布工具当前先提供横向对齐与水平分布，尚未扩展到更完整的版式系统。
- 当前能力定位仍是“文档内嵌 SVG 的轻量结构化编辑”，不是完整通用矢量设计器。

## 补充更新（2026-04-20 11:00）

### 10. HTML 预览中的 SVG 完整编辑器入口已修通

- 之前安装版里出现“SVG 能选中、能拖动，但没有完整编辑器入口”的问题，根因是父层 `HtmlPreview` 只接了 `svgSelection`，没有正式消费 `open-svg-editor`。
- 本轮已将 `open-svg-editor` 正式接通到 `HtmlPreview`，并把“预览内轻编辑会话”和“完整 `SvgTextCanvasEditor` 弹层”拆成两套显式状态。
- 现在单击 SVG 元素仍保留预览内轻编辑；选中后会显示明确的 `Edit SVG` 入口，双击已选中的 SVG 也可直接进入完整编辑器。

### 11. 轻编辑草稿与完整编辑器草稿现已保持一致

- 预览内拖动或 resize 产生的 `svgPreviewPatch` 会继续并入当前轻编辑草稿。
- 打开完整编辑器时，父层会优先使用当前同一 `svgLocator` 的 draft items 作为初始状态，不再退回到最初采集值。
- 这意味着用户可以先在预览里快速挪位置，再进入完整编辑器精修文字、路径或多选内容，而不会互相覆盖。

### 12. 预览宿主脚本已补上显式入口与快捷路径

- SVG 选中覆盖层现已增加 `Edit SVG` 按钮，按钮直接锚定在当前选中框附近。
- 双击已选中的 inline SVG 时，会直接发送完整编辑器请求。
- 点击空白区域、按 `Esc`、或切换出当前 SVG 选择时，这个入口会随选中态一起收起。

## 补充更新（2026-04-20 13:40）

### 14. `Edit SVG` 入口点击稳定性已修复

- 预览中的 `Edit SVG` 按钮已从依赖 `click` 的路径改为更直接的 `pointerdown` 触发。
- 这修复了安装版/WebView 环境里“按钮能看到，但点击没有反应”的问题。
- 当前仍保留两段式交互：单击 SVG 元素做轻编辑，显式点 `Edit SVG` 或双击已选中的 SVG 才进入完整编辑器。

### 15. `Shift+点击` 多选已恢复稳定

- 之前画布 item 同时在 `pointerdown` 和 `click` 两个阶段处理选择，`Shift+点击` 会先加选再被 click 切回，表现为多选失效。
- 现在选择逻辑已经收敛到单一事件通道，多选、批量样式和批量移动会按预期工作。

### 16. 连接线更容易选中了

- 预览宿主新增了连接线近点命中兜底：即使原始 target 落在 SVG root 或留白层，只要点击点在连接线附近，也会优先选中最近的连接线元素。
- 完整编辑器中的 connector item 同步增加了放大的不可见热区，细线、短线和贴边连接线会更容易被选中。
- 这次只放大 connector 的热区，没有扩大普通 shape 的误触范围。

## 补充更新（2026-04-20 14:15）

### 18. 安装包里 `Edit SVG` 可见但点不动的根因已按结构修复

- 进一步排查后确认，高概率问题不在父层 `open-svg-editor` 消息链，而在 iframe 宿主的 SVG 选中覆盖层结构。
- 之前 `Edit SVG` 按钮和 resize handle 都挂在一个全屏 `pointer-events: none` 的 overlay 下，只依赖子节点 `pointer-events: auto` 恢复点击；这在单测里能过，但在 WebView/Chromium 类安装包环境里并不总是稳定。
- 本轮已把 overlay 重构为“纯视觉层 + 独立交互控件”：选中框继续在透传层里渲染，按钮和 8 个手柄则独立挂到 `document.body`，不再处于任何 `pointer-events: none` 祖先之下。

### 19. `Edit SVG` 已恢复标准按钮语义

- `Edit SVG` 入口已恢复为标准 `click` 触发。
- 这样安装包里会走更接近普通 fixed 按钮的命中与辅助功能链路，同时仍保留“单击轻编辑、点击按钮进入完整编辑器”的两段式体验。

### 20. 结构回归已补齐

- 已新增宿主脚本回归，明确断言 `Edit SVG` 按钮与 resize handle 都不在 `[data-mdpad-svg-selection-overlay]` 下面。
- 同时保留“点击按钮会发 `open-svg-editor`”的行为断言，防止未来回归到原先那种 JSDOM 看起来没问题、安装包里却点不动的结构。
