# 修正 16:9 Slide 内容区比例模式

## 变更摘要
- `16:9 Slide` 现在约束应用主内容区，而不是 HTML 预览内部 iframe 舞台。
- 主内容区为标题栏下方、状态栏上方区域；标题栏和状态栏在 16:9 模式下保持可见。
- 开启 16:9 模式后，窗口 resize 会根据主内容区尺寸自动纠正窗口 inner size，使主内容区维持 16:9。
- 增加 `Esc` 退出 16:9 模式，以及顶部 hover 显示的 `退出 16:9 Slide` 按钮。

## 实现说明
- App 层新增应用级 16:9 状态，并通过 `.is-app-slide-aspect` 标记根节点。
- `HtmlPreview` 移除 `isSlideAspect` 渲染职责，只继续填满外部容器。
- `windowPreset` 新增主内容区 16:9 窗口尺寸计算 helper，统一处理 chrome offset、work area 夹紧和 resize 方向。
- 全屏、最大化和 `40% x 90%` preset 与 16:9 模式互斥，执行前会退出 16:9 模式。

## 强约束补充
- 点击 `16:9 Slide` 后，主内容区默认高度改为当前显示器 workArea 高度的 60%，再按 16:9 计算宽度。
- 16:9 模式退出按钮已移动到主内容区右上角，不再覆盖 titlebar。
- 16:9 模式下会临时关闭原生窗口 resize，并在主内容区边缘提供透明 resize handles，由应用按 16:9 主动计算窗口尺寸。
- `onResized` 仍作为兜底校正，用于处理系统级或非预期 resize 漂移。
