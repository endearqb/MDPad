# 2026-04-26 21:00 富文本表格混合粘贴崩溃修复

## 背景

从网页复制“标题/正文 + 表格”的富文本内容粘贴到编辑器时，应用可能显示 `MDPad startup failed`，错误为 `Position 23 outside of fragment (<paragraph>)`。单独粘贴表格或单独粘贴标题文本不会触发。

根因是含 `text/html` 的富文本粘贴会先进入 ProseMirror 默认 HTML parse slice 流程，早于现有 `handlePaste` pipeline；外部网页表格和块内容混合时可能形成不稳定 slice，异常最终冒泡到全局 fatal screen。

## 本次改动

- 新增 `richHtmlTables` 粘贴 helper，用于识别高风险富文本表格 HTML。
- 对外部表格 HTML 做 DOMParser 清洗：移除脚本、样式、事件属性、`colgroup` 等噪声。
- 规范表格单元格：空单元格补段落，直接文本或 inline-only 单元格包进 `<p>`，简单 `<div>` 单元格转换为段落。
- 在 `MarkdownEditor` 的 `handleDOMEvents.paste` 早期接入该 helper，先复用现有媒体/Markdown pipeline，再对高风险 HTML 执行受控 `pasteHTML`。
- 粘贴成功后调度 `fixTables` 修复表格不变量；若 HTML 粘贴失败，则降级为纯文本粘贴。
