# MDPad 更新说明

- 生成时间：2026-04-20 15:25（Asia/Shanghai）
- 目标版本：`0.2.6`
- 文档类型：仓库内更新记录

## 更新概览

本轮更新聚焦 HTML 预览层的可编辑能力扩展，尤其是 SVG 的预览内直编、完整画布编辑器接入、连接线与路径编辑，以及图表与目录锚点交互稳定性修复。同时对编辑器启动包体进行了拆分优化，并补充了纯文本粘贴 Markdown 识别能力。

## 本轮重点更新

### 1. HTML 预览中的 SVG 编辑能力大幅增强

- HTML 预览已支持 SVG 预览内选中、拖拽、基础缩放与右侧属性面板联动，保留最小 patch 回写策略。
- `HtmlPreview` 已正式接入完整 `SvgTextCanvasEditor`，支持从轻编辑会话进入完整编辑器，并继承当前草稿状态。
- 画布编辑器补齐了 `text` / `tspan`、`line`、`path`、`polyline`、`polygon` 等常见元素的结构化编辑能力。
- 新增多选、框选、批量移动、批量样式应用、撤销重做、视口缩放/平移，以及常见贝塞尔路径的锚点与控制柄编辑。
- 连接线支持更强的命中、端点吸附与正交路由回写，适合文档内嵌流程图的轻量结构化修改。

### 2. 预览层交互链路稳定性修复

- SVG 的 `Edit SVG` 入口已完成结构性修复，交互控件从透传 overlay 中拆出，避免桌面安装包/WebView 环境里“按钮可见但点击无效”。
- `Shift+点击` 多选逻辑已收敛到单一事件通道，修复了加选后又被 click 阶段反向切回的问题。
- 连接线在 HTML 预览宿主与完整 SVG 编辑器里都补充了更稳的命中热区，细线、短线和贴边连线更容易被选中。
- HTML 预览里的目录锚点改为“两段式”处理，既阻止错误导航，又保留文档自身目录脚本和嵌套滚动容器的滚动能力。

### 3. 图表编辑入口与预览交互更克制

- 图表交互从“点击容器直接打开编辑器”改成“两段式”：先在真实渲染面上命中图表，再显示显式“编辑图表”入口。
- Chart.js / ECharts 都收紧为真实渲染节点命中，图表容器留白、标题和外层间距不再误触发编辑。
- SVG 与图表的入口都更明确，预览层从“容易误触发”转向“先选择、再进入精修”。

### 4. 编辑器包体与加载路径优化

- 版本号已同步到 `0.2.6`，并同步更新 `package.json`、Tauri 配置与 Rust crate 版本。
- 纯文本导出工具已从富文本导出模块中拆出，避免首页入口静态依赖 TipTap 导出链路。
- `SourceEditor` 改为“核心编辑器 + 按需语言包”模型，`html/javascript/json/python` 语言支持通过动态加载分块引入。
- `vite.config.ts` 的编辑器相关手动分包规则已重整，减少首页主包对富文本和语言扩展的提前绑定。

### 5. Markdown 兼容性与输入体验补充

- 新增纯文本粘贴 Markdown 识别逻辑：当剪贴板只提供 `text/plain`、且文本明显符合 Markdown 结构时，会按 Markdown 渲染结果插入而不是按普通纯文本落入编辑器。
- 新增 Markdown 锚点路由相关回归测试，覆盖内部目录、CJK 标题锚点等常见场景。
- 本轮继续补充 HTML 预览、SVG 会话、图表入口、锚点滚动、拆包与 Markdown 处理相关测试。

## 当前定位

- 当前 SVG 能力定位仍是“文档内嵌 SVG 的轻量结构化编辑”，不是完整通用矢量设计器。
- 正交连接线路由聚焦同一 SVG 内的流程图类用法，不做更复杂的全局拥塞求解。
- 含 `A` 弧线命令的路径仍以文本级编辑为主，不提供不稳定的可视手柄。
