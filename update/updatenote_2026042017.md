# MDPad 更新说明

- 生成时间：2026-04-20 17:30（Asia/Shanghai）
- 目标分支：`main`
- 文档类型：仓库内更新记录

## 更新概览

本轮更新继续收口 HTML 预览层的 SVG / Chart 编辑体验，重点放在“入口稳定、交互克制、草稿可继承”这三件事上。除了修复 `Edit SVG` 的父层入口链路和消息时序问题，本轮还把 `Edit SVG` / `Edit Chart` 正式并入 HTML 预览右键菜单，并保留原有浮动按钮作为并行入口。

## 本轮重点更新

### 1. HTML 预览右键菜单已接入 SVG / Chart 编辑入口

- HTML 预览 iframe 发往父层的 `open-context-menu` 消息已升级为“坐标 + 上下文请求”模型。
- 右键普通区域时，菜单继续只显示导出项。
- 右键命中真实 chart 渲染面时，菜单会追加 `Edit Chart`。
- 右键命中 inline SVG 任意区域时，菜单会追加 `Edit SVG`。
- 两个菜单项都会直接复用现有完整编辑器链路，不新增另一套编辑实现。

### 2. SVG 入口现在支持“浮动按钮 + 右键菜单”并存

- 之前 `Edit SVG` 已迁到父层 `HtmlPreview` 渲染，本轮继续保留这条显式入口。
- 打开右键菜单时，不再顺手清掉 `Edit SVG` / `Edit Chart` 浮动按钮。
- 用户现在可以按偏好选择入口：点浮动按钮，或直接从右键菜单进入编辑器。
- 这次采用的是“共存”方案，不是替换方案，因此不会打断现有使用习惯。

### 3. SVG 草稿继承链路继续加固

- iframe 宿主脚本现在会统一通过 `buildSvgEditorRequest(...)` 收集完整 SVG 编辑请求。
- 父层通过 `sync-svg-session` 回灌到 iframe 的 inline draft，会体现在右键菜单打开的完整编辑请求里。
- 对同一 SVG 而言，用户可以先在轻编辑里拖动 / resize，再通过右键菜单进入完整编辑器，而不会丢掉当前草稿。
- 同时，父层对 `svgSelection` / `svgSelectionFrame` 仍保留抗竞态兜底，避免消息顺序抖动导致入口缺失。

### 4. `HtmlPreview` 已继续推进瘦身

- SVG inline / canvas 会话构建、patch 合并和差异提取逻辑已拆到独立 helper 文件。
- `HtmlPreview.tsx` 当前更集中在父层状态编排、消息消费和入口渲染，单文件职责比之前更清晰。
- 这属于一次保守拆分，优先减轻热路径组件复杂度，没有在本轮继续大范围拆宿主脚本。

### 5. 相关回归测试已同步补齐

- `HtmlPreview.test.ts` 已覆盖：
  - 普通右键菜单只显示导出项
  - chart 上下文菜单打开图表编辑器
  - svg 上下文菜单打开 SVG 编辑器
  - 菜单打开后浮动入口仍可继续使用
- `htmlPreviewDocument.test.ts` 已覆盖：
  - context menu payload 解析
  - 普通区域 / chart / inline SVG 三类右键消息
  - 同一 SVG 已存在 draft 时，菜单请求继续复用最新草稿

## 当前定位

- 当前 SVG 编辑仍定位为“文档内嵌 SVG 的轻量结构化编辑”，不是完整通用矢量设计器。
- 图表入口仍保持克制，只在真实 chart 表面命中时才提供编辑项，不会在容器留白或标题区域误触发。
- 右键菜单的增强目前仅作用于 HTML 预览，不扩展到 Markdown 富文本视图或其它编辑面板。
