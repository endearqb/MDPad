# HTML Preview Editing Hardening

## Summary
- Hardened HTML preview editing so inline text, SVG, and chart writes now fail safely when the source changed instead of patching stale locators blindly.
- Tightened preview URL handling by stripping `javascript:` from rewritten resource URLs while keeping link interception behavior inside the sandboxed iframe.
- Upgraded chart editing from “edit existing numbers only” to a stricter structured editor with label/series add-remove-reorder support and adapter-backed config preservation.

## Highlights
- Added structured patch results in [htmlPreviewEdit.ts](/D:/MyProject/MDPad/src/features/editor/htmlPreviewEdit.ts) with explicit failure reasons such as `LOCATOR_NOT_FOUND`, `TAG_MISMATCH`, `SOURCE_CHANGED`, and `INVALID_CHART_MODEL`.
- Added source verification fields across preview edit requests:
  `currentText` for inline text,
  `sourceSnapshot` for SVG items,
  `sourceFingerprint` for chart bindings.
- Updated [HtmlPreview.tsx](/D:/MyProject/MDPad/src/features/editor/HtmlPreview.tsx) to surface patch failures through an in-app banner instead of only logging them.
- Added [chartAdapters.ts](/D:/MyProject/MDPad/src/features/editor/chartAdapters.ts) to normalize and serialize Chart.js / ECharts models while preserving adapter-specific `sourceConfig`.
- Added [svgEditCapabilities.ts](/D:/MyProject/MDPad/src/features/editor/svgEditCapabilities.ts) and started routing SVG inspector/editor UI through explicit capabilities rather than repeated raw tag checks.
- Expanded [ChartDataEditor.tsx](/D:/MyProject/MDPad/src/features/editor/components/ChartDataEditor.tsx) to support label/series creation, deletion, renaming, reordering, and strict `Number(trimmed)` validation.

## Notes
- This cycle intentionally stays at translate-level SVG transform editing and does not add `<g>` editing, matrix transforms, or CSS-variable-preserving style authoring yet.
- The oversized HTML preview / SVG editor modules now have better internal contracts, but the full module split remains a follow-up step.

## Follow-up Fix
- Added a dedicated inline-SVG drag commit path so preview dragging no longer dies at the “draft only” stage: the iframe still streams `svg-preview-patch` during drag, but now emits a single `svg-commit-patch` on `mouseup` that the parent preview persists back into the HTML source.
- Hardened SVG context-menu entry detection with active-selection fallback, so `Edit SVG` still appears when the current selection is valid even if the raw `event.target` is not resolved back to the inline `<svg>` root.
