# MDPad Update Note - 2026-04-21 11:00

## HTML / Slide Visual Editing Phase 1

- Reworked `HtmlPreview` so visual edits no longer rebuild the iframe on every HTML string change. Live DOM updates now happen inside the sandboxed preview first, while source HTML is patched in the background through `commitVisualHtmlChange(...)`.
- Added a general HTML visual editing bridge for sandboxed previews, including element selection, element frame reporting, live element patch application, drag commit messages, surface mode synchronization, and slide state updates.
- Introduced a first-pass HTML element inspector that edits common text, typography, and layout fields without forcing computed style back into source HTML.

## Slide Workflow

- Added preview-local surface modes inside `HtmlPreview`: `preview`, `visual-edit`, `slide-reading`, and `slide-present`, while keeping the app-level HTML view mode as `preview | source`.
- Added heuristic slide detection plus manual “Treat as Slides” control with path-based preference persistence for saved files and in-memory fallback for unsaved HTML.
- Added reading/present workflows for slide-like HTML, keyboard slide stepping, and Tauri fullscreen support for present mode.

## Source Patching and Safety

- Extended `htmlPreviewEdit.ts` with generic HTML element patch types and helpers, including text, attribute, inline style, and layout patching.
- Marked source-backed DOM nodes inside the preview host script so runtime-generated nodes can be selected but won’t be written back into source HTML by mistake.
- Kept existing SVG and Chart editing flows intact while moving them onto the new “visual commit without iframe reload” architecture.

## Structural Cleanup

- Added stable extraction points under `src/features/editor/html-preview/` for preview document building and bridge APIs, making future decomposition of `htmlPreviewDocument.ts` safer.
- Added the new `html-visual/` and `slides/` modules to keep generic element editing and slide-specific behavior out of the main preview component.
